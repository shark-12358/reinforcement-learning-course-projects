019	a9	摘要
020		本期末專題以RL裡面的PPO、DDPG兩種演算法訓練邊緣攀爬機器人達成目標，演算法主要是訓練補償策略，使機器人能在原有攀爬策略中，靠著補償模型實現不同階梯幾何中穩定向前移動並完成攀爬。現階段完成 PPO、DDPG 訓練與結果整理，包含單一環境訓練、多環境訓練、摩擦係數隨機化、感測雜訊與 PPO 端觀測濾波。
021	a9	研究動機與目的
022		邊緣攀爬對小型機器人而言具有很多的非線性接觸、姿態穩定與關節協調問題。若完全以人工設計軌跡控制，對環境變化的適應能力有限；若完全由強化學習從零探索，訓練難度跟成本都過高。本專題採用參考預設軌跡加補償學習的方式：環境提供一組設計好的關節參考軌跡，PPO policy 學習疊加在參考命令上的殘差動作，藉此降低探索難度並保留學習彈性。
023	a9	實驗方法
024		一、環境設定
025	afc	環境包含 `env1` 至 `env5`。其中 `env1` 至 `env4` 用於訓練與評估，`env5` 用於未見環境測試。成功條件為前端位置超過目標區域且機器人未跌倒。
026		二、觀測設計
027		觀測包含關節角度、關節速度、機身姿態、夾爪姿態與步態相位等資訊。Policy 不直接接收環境編號、階梯幾何或摩擦係數，因此若模型能在不同環境中成功，代表它必須從機器人狀態與接觸後的動態反應推估合適控制。
028		三、獎勵設計
029		這裡的 reward是讓 agent 在隨機摩擦、雜訊、濾波環境下學習，同時 reward 加入很多「穩定性懲罰」，讓策略不要只追求往前衝，而是要穩定、平順、姿態正確地爬上 ledge。有沒有穩健控制並不影響reward計算，只是影響練環境乾不乾淨而已。
030		獎勵細項
031		內容
032		1. 前進獎勵
033		只要機器人前端往前移動，就給正 reward。
034		2. 成功獎勵
035		成功條件不是只有到達目標 x，還要姿態穩定。
036		3. 速度獎勵
037		如果越早成功，success_speed_remaining_fraction 越大，額外 bonus 越高。
038		4. 時間懲罰
039		每一步都扣一點，避免 agent 拖太久。
040		5. Pace 懲罰
041		如果時間已經過很多，但前進進度太少，就扣分。
042		也就是說它鼓勵機器人在合理時間內持續推進，不要停在原地或慢慢磨。
043		6. Action 懲罰
044		不要對參考軌跡做太大的修正。
045		7. Smoothness 懲罰
046		這一步 residual action 和上一歩 residual action 的差。
047		8. 關節速度懲罰
048		懲罰平均關節速度太高。這也是穩健設計的一部分，避免快速甩動導致接觸不穩定。
049		9. Base 穩定性懲罰
050		希望 agent 不能用歪斜、橫向漂移的方式硬爬。
051		10. Gripper 穩定性懲罰
052		希望 gripper 接觸姿態穩、左右一致、不要歪抓。
053		11. Fall / Timeout 懲罰
054		跌落懲罰很重，timeout 懲罰較輕。

056		四、穩健控制
057		在本次的實驗中，為了更貼近現實生活，而不是單純的紙上談兵，我們模擬了現實生活中會遇到的雜訊進去，例如Sensor noise、摩擦力。
058		在最基本的訓練上，使用了多環境訓練，讓agent同時學會處理多種情況。Single-env training 是在單一階梯環境中訓練 policy，主要用來觀察演算法是否能學會特定環境下的攀爬策略。Multi-env training 則是在多個階梯環境中交替訓練同一個 policy，使模型不能只記住單一地形，而必須學習較通用的控制修正能力，因此更適合用來測試策略的泛用性與強健性。
059		Sensor noise流程為下，PyBullet 真實狀態→ clean observation→
060		加入 sensor noise→ noisy observation→ PPO observation filter→ normalization→ policy network
061		加入Sensor noise的部分只有「感測器資訊」，「內部已知資訊」不加入，例如:previous residual action、reference phase、reference action。
062		摩擦力則是每次reset時會隨機抽出數值，阻止接觸面切向滑動、旋轉滑動、滾動滑動。直接經由PyBullet作用。

064		五、演算法差別
065		本次實驗採用兩種演算法PPO和DDPG，兩者的都是以AC演算法為基底去延伸出來的算法。
066	a9	PPO
067		PPO，全名 Proximal Policy Optimization。，在AC演算法之上加入核心觀念「策略可以更新，但不能一次改太大」，用以前訓練模型為例，例如A2C都很常因為一次的更新而將整個模型推壞，因此PPO的核心觀念就是防止某一次的大更新推壞模型。
068	a9	DDPG
069		DDPG，全名 Deep Deterministic Policy Gradient。它也是 Actor-Critic，但跟 PPO 最大不同是：DDPG是決定性策略，也就是說，它不抽動作，而是直接輸出動作值。

094		連續控制、機械臂、車輛控制
095		一般 Actor-Critic 方法多採用隨機策略，其 Actor 輸出的不是單一確定動作，而是動作的機率分布。若動作空間為離散形式，例如向左或向右移動，Actor 會輸出各個動作被選擇的機率，再依據該機率分布進行抽樣。若應用於連續動作空間，Actor 則通常會輸出分布參數，例如平均值與標準差，再從該分布中取樣得到實際動作。此方法的優點是具有較自然的探索能力，因為策略本身包含隨機性，即使目前模型傾向選擇某一動作，仍有機會嘗試其他動作，因此較不容易在訓練初期過早陷入局部最佳解。不過，隨機策略在連續控制問題中需要額外設計機率分布與探索幅度，若標準差設定不佳，可能造成動作過於混亂或探索不足。
096		DDPG 則採用決定性策略，其 Actor 會直接根據目前狀態輸出一組連續動作值。例如在車輛控制或機械臂控制中，Actor 可以直接輸出油門大小、方向盤角度、馬達力矩或關節角速度等控制量。這種方式較符合實際控制系統的概念，因為控制器通常也是根據目前狀態直接產生控制輸出。因此，DDPG 特別適合處理連續動作空間，且不需要像離散動作方法一樣枚舉所有可能動作。此外，DDPG 屬於 off-policy 方法，可搭配經驗回放機制重複使用過去收集到的資料，因此在資料利用率上具有優勢。
097		然而，DDPG 的缺點是探索能力並非策略本身自然產生，而是需要額外在 Actor 輸出的動作上加入雜訊。若雜訊過大，動作可能過於不穩定；若雜訊過小，則可能導致探索不足。此外，DDPG 的學習效果高度依賴 Critic 對 Q 值的估計，若 Critic 評估不準確，Actor 可能會朝錯誤方向更新，使訓練過程變得不穩定。因此，雖然 DDPG 在連續控制問題上具有直覺且高效率的動作輸出方式，但相較於一般隨機策略的 Actor-Critic 方法，其訓練穩定性通常較差。
098		整體而言，一般 Actor-Critic 的優勢在於探索能力較自然、訓練過程相對穩定，適合離散動作或需要保持探索性的問題；DDPG 的優勢則在於能直接輸出連續控制量，較適合車輛、機械臂與馬達控制等連續動作任務。若從控制角度來看，一般 Actor-Critic 較像是根據機率嘗試不同動作，而 DDPG 則較像是一個直接輸出控制命令的智慧控制器。
